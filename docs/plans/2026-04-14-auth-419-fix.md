# Auth 419 Fix Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Remove the recurring `419 Page Expired` failures from login and register by fixing the shared session/CSRF layer and verifying the full auth form flow live.

**Architecture:** Reproduce the failure first with live form submissions so the cause is confirmed, then correct the shared session-cookie configuration or request host mismatch that invalidates CSRF tokens between GET and POST. Keep the fix centralized in config/runtime behavior rather than form-specific hacks, then verify both login and register with browser automation.

**Tech Stack:** Laravel 8, Blade, session/cookie middleware, Docker/Nginx, Playwright

---

**Files:**
- Modify: `tests/playwright/02-auth.spec.ts`

**Step 1:** Add failing live tests that submit the login and register forms normally.

**Step 2:** Run only those tests to confirm the current failure mode.

### Task 2: Fix shared CSRF/session behavior

**Files:**

- Modify: middleware/config files only if the confirmed cause requires it

**Step 1:** Make cookie domain, secure cookie, same-site, and path handling environment-driven where needed.

**Step 2:** Ensure the app can serve GET and POST on the same effective host without cross-host cookie loss.

**Step 3:** Avoid form-level workarounds unless reproduction proves a form-specific issue.
