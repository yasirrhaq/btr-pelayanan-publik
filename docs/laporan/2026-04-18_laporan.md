# Laporan Progress 2026-04-18

## Ringkasan

Pada update ini, pekerjaan difokuskan pada dua area besar:

1. **Halaman Publik**
- melakukan redesign halaman publik berdasarkan arahan dan style **TopDown**
- merapikan struktur navigasi publik
- menambah halaman publik baru yang sebelumnya belum tersedia
- menghubungkan konten publik dengan data dari backend admin

2. **Halaman Admin**
- merapikan CMS/admin panel agar lebih konsisten
- memperbaiki editor konten
- menyamakan pola wizard/tab di beberapa modul
- memperjelas pemisahan data dan alur kerja admin

## A. Progress Halaman Publik

### 1. Redesign Halaman Publik Berdasarkan TopDown

- Beberapa halaman publik telah disesuaikan ulang mengikuti pendekatan desain **TopDown** agar tampil lebih modern, rapi, dan konsisten.
- Struktur informasi publik dibuat lebih mudah dipahami user.
- Topbar publik diperbaiki agar lebih terarah dan lebih sesuai dengan struktur menu yang disepakati.

### 2. PPID Publik

- Dibuat halaman publik PPID yang lebih rapi dan terhubung ke backend admin.
- Struktur publik PPID sekarang terdiri dari:
  - `/ppid`
  - `/ppid/kebijakan-ppid`
  - `/ppid/info-berkala`
  - `/ppid/info-serta-merta`
  - `/ppid/info-setiap-saat`
- Menu topbar publik untuk PPID juga sudah diarahkan ke halaman publik yang benar.

### 3. Dokumen Publik

- Ditambahkan halaman publik:
  - `/dokumen`
- Halaman ini dipakai untuk menampilkan dokumen/lampiran dari sumber yang saat ini sudah tersedia, seperti:
  - lampiran pengumuman
  - dokumen galeri
  - file dokumen yang muncul dari konten berita bila ada

### 4. Renstra / Rencana Strategis

- Dibuat halaman publik baru:
  - `/renstra`
  - `/renstra/{slug}`
- Topbar publik **Rencana Strategis** sudah diarahkan ke halaman publik Renstra, bukan ke dashboard admin.
- Struktur route publik dan admin juga sudah dipisahkan agar tidak bentrok.

### 5. Halaman Maskot Balai Teknik Rawa

- Ditambahkan halaman publik baru:
  - `/maskot-balai-teknik-rawa`
- Halaman ini mengambil data dari pengelolaan konten maskot pada admin identitas/profil singkat.

### 6. Halaman Informasi Pegawai

- Halaman publik informasi pegawai diperbaiki agar tidak lagi hanya menampilkan gambar mentah.
- Sekarang data pegawai ditampilkan dalam format kartu informasi yang lebih rapi dan lebih mudah dibaca.
- Struktur data publik pegawai sekarang mengikuti data terstruktur dari admin.

### 7. Struktur Menu Topbar Publik

- Topbar publik sudah disusun ulang mengikuti struktur menu yang diinginkan:
  - Profil
  - Layanan
  - Publikasi
  - Saran dan Pengaduan
- Dropdown dan subsubmenu diperbaiki supaya:
  - hanya muncul saat hover
  - lebih menyatu secara visual
  - tidak terasa terpisah atau mengambang

### 8. Integrasi Halaman Publik dengan Backend Admin

- Konten publik yang sudah terhubung ke backend admin meliputi:
  - PPID
  - Renstra
  - Profil Singkat / Identitas
  - Maskot
  - Informasi Pegawai
- Ini membuat update konten dari admin langsung berdampak ke tampilan publik.

## B. Progress Halaman Admin

### 1. Editor Konten Admin

- Editor rich text pada beberapa modul admin telah diganti ke **Jodit Editor**.
- Editor ini digunakan agar pengalaman input konten lebih nyaman bagi admin.
- Upload gambar tetap terhubung dengan endpoint Laravel yang sudah ada.

### 2. Modul PPID Admin

- Modul PPID diperbaiki agar lebih konsisten dan lebih stabil.
- Struktur tab PPID admin sudah terhubung dengan halaman publik PPID.
- Mapping data PPID admin dan publik sudah dipusatkan agar tidak berbeda antar sisi.

### 3. Profil Singkat / Identitas Admin

- Halaman `/dashboard/profil-singkat` diperbaiki menjadi model wizard/tab untuk:
  - Tentang Kami
  - Sejarah
  - Visi & Misi
  - Tugas & Fungsi
  - Maskot
- Tampilan wizard/tab dibuat lebih rapi dan konsisten.
- Konten maskot juga sudah dipisahkan dengan lebih jelas sesuai kebutuhan admin.

### 4. Informasi Pegawai Admin

- Modul informasi pegawai dikembangkan dari sekadar gambar menjadi data pegawai yang terstruktur.
- Field yang sekarang dikelola meliputi:
  - nama
  - nip
  - jenis kepegawaian
  - pangkat/golongan
  - jabatan
  - instansi
  - email
  - urutan
- Admin sekarang bisa menentukan **urutan/order** pegawai.
- Ditambahkan juga opsi untuk menghapus foto tanpa menghapus data pegawai.

### 5. Struktur Organisasi Admin

- Halaman `/dashboard/struktur-organisasi` diperbaiki agar lebih konsisten dari sisi:
  - judul halaman
  - alignment tabel
  - tombol aksi
  - tab SDM
- Sidebar admin juga sudah diatur ulang sehingga **Struktur Organisasi** berada di atas **Informasi Pegawai**.

### 6. Penyatuan Wizard/Tab Admin

- Wizard/tab admin sekarang diseragamkan mengikuti gaya PPID.
- Ini sudah diterapkan pada beberapa modul seperti:
  - PPID
  - Profil Singkat
  - Struktur Organisasi
  - Informasi Pegawai
  - Galeri
- Ukuran visual tab juga sudah diperkecil sedikit agar lebih ringan dan lebih rapi.

### 7. Renstra Admin

- Modul admin yang sebelumnya masih bernama atau terasa seperti **Karya Ilmiah** sekarang sudah diarahkan menjadi **Renstra / Rencana Strategis**.
- Label admin, route admin, dan alur penggunaannya sudah disesuaikan agar lebih sesuai dengan kebutuhan client.

### 8. Penyempurnaan UI/UX Admin dan Pelanggan

- Header/topbar admin dan pelanggan telah dirapikan agar lebih konsisten.
- Flow profil pelanggan dipusatkan di area pelanggan agar user tidak bingung dengan dua jalur profil yang berbeda.
- Sidebar, logout, dropdown profil, dan beberapa tabel juga sudah diperbaiki tampilannya.

## Dampak Hasil

- Website publik sekarang lebih siap dipresentasikan ke client karena struktur halaman dan navigasi sudah lebih jelas.
- Admin panel menjadi lebih mudah dipakai untuk mengelola konten.
- Konten publik dan backend admin sekarang lebih terhubung dan lebih konsisten.
- Beberapa modul lama yang sebelumnya masih bercampur istilah lama/baru sudah mulai dirapikan.

## Catatan Lanjutan

Beberapa hal yang masih bisa dilanjutkan pada tahap berikutnya:

- finalisasi istilah lama yang masih tersisa di level kode internal
- validasi data pegawai dengan data master terbaru client
- penyempurnaan struktur dokumen publik bila sumber dokumen bertambah
- migrasi asset frontend dari pola CDN ke asset build bila nanti dibutuhkan
