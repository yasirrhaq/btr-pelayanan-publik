# Operations Docs

`docs/operations/` is the repo working area for engineers and agents.

## Contents

- `WORKFLOW.md`: how work should be executed in this repo
- `RULES.md`: concise working constraints and defaults
- `IMPLEMENTATION_TRACKER.md`: compact status board for shipped and in-progress features
- `MEMORIES.md`: project working memory and reminders
- `sessions/`: handoff notes and session-specific operational logs

## Rule

Do not put durable feature documentation here unless it is specifically an operational handoff.
Use `docs/implementation/` for long-lived feature writeups.
